Sprint 4 - Technical Blog

Here are some of my sprint 4 struggles and resolutions

Got blocked on a simple problem

So I’ll start of by explaining that some programming languages are object orientated.

This means that you can assign values to variables, similar to in algebra with x = 2 or y = -12.

You can name these variables many ways, but there are rules and some guidelines.

I found myself stuck on a problem for about twenty minutes, I was going in circles.

Posted to the slack channel, as soon as I did, I noticed the problem was a misspelled variable.

One letter, twenty minutes of pain.

I used console logging, reading error messages and googling to be honest none of that helped.

What would have worked is if I stepped away for a moment or asked a colleague.

This was super frustrating and probably even more so when I figured out the issue


Solved a problem in an elegant way.

There was a problem I was working that tested to see if two variables had the same result.

These variables, let’s call them Adam and Terah, are quite complex and essentially contain a collection of variables. 

I released instead assigning various values and typing everything out again I can simply make Adam = Terah

I started out with pseudocode, which helped me understand and visualise these complicated variables.

Also a whole lot of console logging to understand where I was going wrong.

This problem was a confusing as I tried the other method and it didn’t work.

When I solved the problem, the satisfaction was very real.


Reflect on how confident you feel using the problem solving techniques and process:

Pseudocode

Fairly confident with my pseudocode. Used it a few times throughout this sprint and it seemed to help my understanding from the get go

Trying something

I did a lot, even going back and doing problems with different methods. I feel quite confident with this as there’s a lot of ways to skin a cat

Rubber ducky method

Not super confident with this method, just haven’t practiced it enough. But it is a very elegant idea and I would like to try more of it

Reading error messages

Did a lot of this, I feel pretty confident with it

Console.logging

This is probably my favourite way to solve problems. I’m extremely visual, so being able to see and compare on the screen helps a lot

Googling

Google is my best friend, there’s so many people running in to the same problems

I did this, but had a eureka moment. I feel comfortable asking, but much prefer grinding through problems myself

Asking coaches for help

I haven’t done this yet, I have a few questions that I’d like to bring up with my coach though.

Improving your process with reflection

I think reflection has helped, taking a step back and understanding my processes. I don’t feel that confident in my reflections, but I definitely think they help even if it’s more subtle.
